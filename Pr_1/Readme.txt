UserRepository працює з даними, зберігаючи їх у файлі JSON.
UserService містить бізнес-логіку додавання, видалення та перегляду користувачів.
Program надає інтерфейс для взаємодії з користувачем через консоль.
Цей додаток підтримує базові операції управління користувачами, перевірку унікальності email, а також збереження даних у JSON файлі.

Тема: Багаторівнева архітектура
Тип архітектури: 3-рівнева (Presentation, Business Logic, Data Access)
Що міститься:
Файл/папка	Опис
Program.cs	Головний файл, де реалізовано інтерфейс користувача (Presentation Layer)
UserService.cs	Логіка управління користувачами (Business Logic Layer)
UserRepository.cs	Збереження/читання з JSON (Data Access Layer)
User.cs	Модель користувача (ім’я + email)
Users.json	Файл, у якому зберігаються користувачі
Readme.txt	Коротке текстове пояснення структури
Project structure	(Можливо, папка з кодом або коментарями — слід уточнити)

Основний функціонал:
Додавання нового користувача

Видалення користувача за email

Перегляд усіх користувачів

Перевірка на унікальність email

Збереження всіх змін у JSON-файлі

Архітектурні особливості:
Кожен шар реалізований окремим класом

Взаємодія відбувається через інтерфейси (це правильно за вимогами ПР)

Реалізована структурована логіка — легко підтримувати та розширювати
